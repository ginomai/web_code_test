iTriage Web Engineer Test

Summary

This code test is given to every prospective new member of the iTriage Javascript development group. The goal of this test is to get a feel for your coding style, and problem solving skills. It is meant to be a simulation of a real day at work, so you are free to use whatever resources you have at your disposal in order to be successful. Please also feel free to bring in UI libraries that you might find helpful in creating a compelling user experience.

Read the rest of this document to get an understanding of the expectations, and be sure to let someone know if you have any questions, run into any odd issues, or need some help getting over a hurdle.

You will be assessed in 3 main areas:

  • Code organization and readability
  • Object oriented design patterns
  • Visual styling

Bonus points for:

  • Use of MV* design patterns
  • Animations and fallbacks
  • Compelling UI/UX

User stories

  • As a user, I would like to be able to browse a list of article categories.
  • As a user, I would like to set my preference for a article category so that I only see articles relating to the category I'm interested in.
  • As a user, I would like to see a list of articles for a category I have selected. As to not get overwhelmed, I would only like to see a listing of 5 articles at a time.
  • As a user, I would like to be able to view the next/previous 5 articles in my preferred category list.
  • As a user, I would like to click on an article and see it's full contents.
  • As a user, I would like some indication that I have read the article so I don't try to read the same article again.
  • As a user, I would like to be able to save an article so I can read it later. I would also like to be able to 'unsave' it when I no longer need it so my list does not get too long. (Use of local broswer storage is fine here)
  • As a user, I would like to be able to view all my saved articles in a list.
